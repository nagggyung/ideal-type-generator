# Sprint 02 — 핵심 기능

## 목표
이상형 조건 입력 → GPT 프롬프트 변환 → DALL·E 이미지 생성 → 결과 화면 표시
전체 핵심 흐름을 완성한다.

## 전제 조건
- sprint-01 완료 (환경변수, Supabase 연동, 인증 기반)

## 범위

### 포함
- [ ] `app/page.tsx` — 이상형 조건 입력 폼 (홈 페이지)
  - 텍스트 입력 영역
  - 예시 키워드 칩 (차분한 분위기, 강아지상, 다정한 느낌 등)
  - "이미지 생성" 버튼
  - 로딩 상태 ("이상형을 그리는 중...")
- [ ] `app/api/generate/route.ts` — 핵심 API
  - 입력 검증 (필수값, 최대 500자)
  - GPT-4o로 한국어 → DALL·E 영문 프롬프트 변환
  - DALL·E 3으로 이미지 생성 (1024×1024)
  - 오류 처리 (타임아웃, Rate Limit 등)
- [ ] `app/result/page.tsx` — 결과 화면
  - 생성된 이미지 표시
  - 변환된 프롬프트 표시
  - "다시 생성하기" 버튼
  - "저장하기" 버튼 (로그인 미시 → 로그인 유도)
- [ ] `export const maxDuration = 30` 설정

### 제외
- 실제 이미지 저장 (DB/Storage)
- 기록 페이지
- 인증 미들웨어

## 완료 기준

1. `npm run verify` 통과
2. 이상형 조건 입력 → 이미지 생성 전체 흐름 동작 확인
3. 오류 발생 시 사용자에게 적절한 메시지 표시 확인
4. "다시 생성하기" 동작 확인
5. `NEXT_PUBLIC_` 환경변수에 `OPENAI_API_KEY` 없음 확인
6. `state/progress.json` sprint-02 완료로 업데이트
7. `docs/DECISIONS.md`에 GPT 프롬프트 전략 기록
